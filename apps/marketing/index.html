Ruler

Visual rule-engine studio. Draw JDM decision graphs, evaluate them, watch executed nodes light up, keep an audit trail. GoRules Zen under the hood.

View on GitHub Run the demo

What’s in the box

Python ruler-engine

Framework-agnostic Python library wrapping GoRules Zen. Pluggable storage and audit sinks. Drop it into FastAPI, Flask, a Lambda, or a script.

pip install ruler-engine

React @ruler/react-editor

React components for the visual editor, live trace overlay, and audit-log table. Typed HTTP client included. Peer deps: React 18+.

pnpm add @ruler/react-editor

HTTP reference service

FastAPI service that wires the Python engine to a small HTTP surface. Java, Go, and other backends consume it via plain JSON.

uvicorn demo_api.main:app

Web reference client

Vite + React demo app showing the editor, trace overlay, and log viewer wired end-to-end. Fork it, or copy the bits you want.

pnpm --filter demo-web dev

Python, minimal

from ruler_engine import RuleEngine, InMemoryStorage, InMemoryAuditSink

engine = RuleEngine(
    storage=InMemoryStorage(),
    audit=InMemoryAuditSink(),
)

engine.save_rule("discount", jdm_json_dict)

outcome = engine.evaluate("discount", {"age": 25, "tier": "gold"})
print(outcome.result)        # -> {"discount": 0.15}
print(outcome.performance)   # -> "312µs"
print(outcome.trace)         # -> dict of executed node outputs

React, minimal

import {
  DecisionGraphEditor,
  LogsViewer,
  createRulerClient,
} from "@ruler/react-editor";

const client = createRulerClient({ baseUrl: "https://api.example.com" });

export function App() {
  const [content, setContent] = useState(myJdm);
  const [trace, setTrace] = useState(null);
  return (
    <>
      <DecisionGraphEditor value={content} onChange={setContent} trace={trace} />
      <LogsViewer client={client} />
    </>
  );
}
